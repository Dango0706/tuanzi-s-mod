# Tuanzi's Mod 1.0.5.1 Changelog

## Fixes

- **Fixed JVM File Lock Exception during Auto-Update Downloads on Windows**:
  - **Issue Details**: Due to Windows' strict file locking mechanism, the active mod `.jar` file remains locked by the running JVM, causing the auto-updater to trigger a `FileSystemException` ("The process cannot access the file because it is being used by another process") when attempting to move or rename the active jar.
  - **Fix**: Implemented a deferred updater sequence for Windows. Instead of modifying the active jar during runtime, the updater downloads the new version as a `.tmp` file and generates a lightweight `update.bat` script in the `mods` folder. A JVM shutdown hook launches the batch file upon game exit; the batch script pauses for 2 seconds to let the JVM fully release file handles, then safely backs up the old jar as `.bak`, applies the new version, and deletes itself, achieving a seamless upgrade flow on Windows.
