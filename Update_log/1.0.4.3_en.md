# Tuanzi's Mod 1.0.4.3 Changelog

## Fixes

- **Fixed an issue where the Warden did not drop 275 experience points upon death**:
  - **Fix**: Changed the target mixin class in [WardenExperienceMixin.java](file:///F:/Development/Java/tuanzis_mods/src/main/java/me/tuanzi/mixin/WardenExperienceMixin.java) from `LivingEntity.class` to `Mob.class`. Because vanilla Minecraft's `Mob.class` overrides the `getBaseExperienceReward` method, the injection on `LivingEntity` was shadowed and bypassed for all mobs (including the Warden). Directing the mixin to `Mob.class` fully resolves this.
  - **Double Optimization for Safety & Logical Precision**: To avoid superclass field resolution issues (`Unresolved field`) when using `@Shadow` on subclasses, we designed and implemented [LivingEntityAccessor.java](file:///F:/Development/Java/tuanzis_mods/src/main/java/me/tuanzi/mixin/LivingEntityAccessor.java).
  - **Bypassing Naming Overwrite Conflicts**: To avoid `InvalidMixinException` classloading failures caused by accessor method names colliding with already-declared methods in the target vanilla class, we renamed the accessor method to a unique, mod-exclusive name: `getTuanziLastHurtByPlayerMemoryTime()`. We registered it properly in `tuanzis_mod.mixins.json`. This implements a 100% strict logic matching to `lastHurtByPlayerMemoryTime > 0` with zero risk of Mixin target conflicts, guaranteeing bulletproof stability.
