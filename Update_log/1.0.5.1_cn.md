# Tuanzi's Mod 1.0.5.1 更新日志

## 修复

- **修复 Windows 平台下自动更新下载时旧 jar 文件被锁定占用的 Bug**：
  - **问题详情**：由于 Windows 系统的底层文件锁机制，JVM 运行期间会对当前加载的模组 `.jar` 文件加锁，导致自动更新进程在尝试移动备份该文件时触发 `FileSystemException` 并抛出“另一个程序正在使用此文件，进程无法访问”的严重错误。
  - **解决方案**：针对 Windows 平台引入了延迟更新机制。自动更新下载时不再直接操作运行中的旧包，而是先将新包下载暂存为临时文件（`.tmp`），并同时在 `mods` 目录中生成一个轻量级延迟替换脚本 `update.bat`；注册 JVM 关闭钩子，在玩家点击重启、当前 JVM 进程优雅退出释放文件锁的一瞬间，后台拉起批处理脚本，延迟 2 秒安全地执行旧包重命名归档（`.jar.bak`）及新包应用，最后脚本自我销毁，从而在 Windows 平台完美实现平滑无感的安全热升级。
