# Tuanzi's Mod 1.0.5 Changelog

## Features

- **Integrated Modrinth Auto-Update Checker**:
  - **Asynchronous Detection on Mod Load**: Asynchronously dispatches a request to Modrinth on client startup to verify if a newer compatible version is available.
  - **Fluid High-Aesthetic Main Menu Dialog**: If an update exists and is not ignored, redirects the user to a sleek dark-themed card panel alongside "Auto Update", "Ignore Updates for a Day" (timestamp saved locally), and "Remind Me Later" buttons.
  - **Aesthetic Hover Bleed-Through Prevention**: Resolved a visual flaw where hovering mouse cursor over gaps in the update card would cause original TitleScreen buttons (like "Singleplayer") to light up (Hover). Implemented via parameter-based coordinate shielding (`-999, -999` passed to parent extractRenderState) to ensure absolute overlay barrier.
  - **Closed-Loop Exit Fix for Ignores & Reminders**: Upgraded current-process notify-blockers (`alreadyNotified`) to the first order of operations. Fixes an infinite-loop logic issue where clicking "Remind Me Later" or "Ignore Updates for a Day" returned to TitleScreen but triggered repeat Mixin init popups, preventing the screen from closing.
  - **Native Markdown Log Rendering**: Built a high-performance, zero-dependency Markdown parser. Translates Modrinth's raw markdown logs (including headings, bullet lists, `**bolding**`, `*italics*`, and `` `gold-orange inline code blocks` ``) into Minecraft's native style Components for a highly polished, aesthetic player experience.
  - **Smooth Scissor-Based Scroll & Premium Scrollbar**: Implemented pixel-level screen Scissor clipping and optimized memory-level text caching. When a changelog is exceptionally long, players can smoothly scroll up and down using the mouse wheel, complete with a beautifully designed **translucent white track and a neon-blue scroller** on the card's right edge for an outstanding interactive experience.
  - **Zero-Deletion Safe Upgrade**: Automatically downloads the latest `.jar` to the mods directory on update confirmation. Adheres strictly to the GEMINI.md mandate of not deleting files by renaming the active old jar to `.jar.bak`.
  - **Reboot Prompt & Safe Exit**: Informs players that a restart is required once the download completes, offering "Restart Now" (safely stops client process) and "Later" options.
  - **Aesthetic Failure Indicator**: Elegantly draws a smooth, hardware-accelerated fade-in/fade-out warning card in the top-right corner of the main menu if network requests fail. Uses soft transparent dark-red glow, displaying error details and silent auto-dismissing after 10 seconds.
  - **Development Sandbox Support**: When running in an IDE development environment, forces update notification checks to bypass all local ignores and once-notified memory flags, including full offline-fallback generation to guarantee update screen pops up seamlessly every time for easier frontend debugs.
  - **Dedicated Server Environment Isolation**: All update-checking, UI popup dialogs, and auto-download installer logic are strictly restricted to the client-side (`EnvType.CLIENT`). Dedicated Servers will completely ignore this update check process, successfully preventing any class-loading conflicts or server runtime crashes.

- **Remade the Linglong Trade Station (Soul Merchant Station) 3D Voxel Model**:
  - **High-Aesthetic Vanilla-Tech Modeling**: Adhered strictly to the `blockbench.md` model standards and traditional Minecraft voxel aesthetics. Avoided smooth slants or rounded edges, restructuring the model into 14 highly detailed elements. Features double heavy baseplates, reinforced two-segment industrial corner pillars, an energy communication spire, and a central suspended quantum fusion core.
  - **Central Floating Fusion Core Repositioning & Collision Safety**: Addressed an awkward clipping flaw in the previous 3D model where the quantum fusion core (`floating_fusion_core`) was stationed at coordinates `[5, 1, 5]` to `[11, 5, 11]` (Y=1-5), severely clipping through the solid baseplate and causing hosted micro-villagers to float. Restructured the core coordinates to `[5, 10, 5]` to `[11, 13, 11]` (Y=10-13, size 6x3x6) to sit elegantly right beneath the ceiling, realizing an authentic "suspended" aesthetic and freeing up block space.
  - **Hosted Micro-Villager Grounding**: Along with the quantum core moving to the ceiling, we overhauled the micro-villager's snapTo coordinate offset inside `SoulMerchantStationBlockEntity.java` from the original `0.3125` (5 voxels) to a finely tuned `0.25` (4 voxels, keeping them precisely 1 voxel above the pedestal surface to negate downscaling render sinking). The hosted micro-villager now stands firmly and naturally on the solid stone pedestal platform, resolving previous foot-sinking and pedestal clipping defects.
  - **Pixel-Perfect 1:1 UV Mapping**: Removed obsolete `"format_version"` definitions in the JSON model to get rid of vanilla resourcepack warnings. Restructured UV mappings for both the `energy_crown` and `floating_fusion_core` into pixel-perfect 1:1 coords (eliminating raw [0,0] corner textures), resolving stretched, blurry panels.
  - **VoxelShape Collision Safety Hardening**: Refactored the corresponding central core bounding box in `SoulMerchantStationBlock.java` to `Block.box(5.0, 10.0, 5.0, 11.0, 13.0, 11.0)`. Furthermore, successfully extracted the out-of-bounds `Block.box(5.0, 16.0, 5.0, 11.0, 18.0, 11.0)` crown box which exceeded the 16.0 voxel hard ceiling. Bounding shapes are now 100% constrained within standard 1x1x1 blocks, completely sealing off any potential physics-driven `IllegalArgumentException` crash loops.
  - **Custom Apocalyptic Cyber Texture**: Utilized AI generation to craft a dark-grey weathered iron sheet texture adorned with industrial rust patches and glowing cybernetic neon-blue circuit paths. Processed via the `process_texture.py` script for lossless background keying, padding-free cropping, and high-fidelity downscaling to a client-approved `32x32` pixel canvas.
  - **Refactored Held and GUI View Scales & Persistent DataGen Bypass Fix**: Specifically targeting the detailed and slightly taller (18 voxels) outline of the 3D Soul Merchant Station, re-tuned display metrics inside the hand-held item model `soul_merchant_station.json` (downscaling hand-held view to `0.25` and GUI slot preview to `0.55`). Addressed a critical Fabric-datagen issue where executing `runDatagen` would persistently overwrite hand-crafted Blockbench block models and customized item display settings with default trivial cubes, leading to massive hand-held shapes and missing voxel elements:
    - **Blockbench Resource Bypass Loader**: Overhauled `ModModelProvider.java` to inject a local file-reading bypass mechanism. During the datagen execution, the generator directly reads the raw Blockbench-exported `block/soul_merchant_station.json` and outputs it losslessly, perfectly preserving the beautiful 14-element hand-crafted structures without breaking compilation or retreating to hardcoded outdated builders.
    - **Item Display Injector Persistence**: Dynamically intercepts the `item/soul_merchant_station` model output in Datagen to append the custom item scaling `display` parameters as high-fidelity GSON structures. Ensures that no matter how many times `runDatagen` is called, the hand-held scaling configurations remain perfectly generated and persistent, permanently solving the giant hand-held scaling bug.
