# Tuanzi's Mod 1.0.4 Update Log

## New Features

- **New Item: Villager Cage**:
  - **Crafting Recipe**: Shaped crafting, pattern as follows:
    ```
    Iron Ingot    Emerald Block    Iron Ingot
    Iron Ingot    Soul Sand        Iron Ingot
    Iron Ingot    Iron Ingot       Iron Ingot
    ```
  - **One-time Use**: Durability of 1. The cage is destroyed immediately after capturing a villager.
  - **Capturing Villagers**:
    - Hold the Villager Cage in your **main hand** and **right-click a villager** to capture it.
    - Plays a **Soul Sand** placement sound and spawns **Ender particles** at the villager's location.
    - The captured villager is removed from the world. All data — **profession, level, trade offers, trade uses, and custom name** — is fully saved in the cage item.
    - The item icon switches to the **filled cage** texture (dual-state texture system).
    - Tooltip displays: `Contains: [Villager Name] - [Profession] (Level X)`
  - **Capture Restriction (Iron Golem Protection — Precise Method B)**:
    - Only triggers if an Iron Golem is in an **active patrolling/guarding state** within 16 blocks of the villager.
    - All guarding Golems will immediately become hostile toward the captor, with a warning message shown.
    - Golems already chasing another target will not be redirected — only those in idle guard mode will switch targets.
  - **Releasing Villagers**:
    - **Sneak + Right-click** on a solid block surface to release. The cage is consumed and the villager spawns above the clicked block with all data fully restored.
    - Release is denied (with a message) if the target location has **no solid block** beneath or **insufficient space** (requires 2 blocks of air above).
    - An empty cage does nothing when sneaking and right-clicking.
  - **Cross-Dimensional Transport**: Villager data is stored entirely in the item's NBT. Release in any dimension without restriction.
  - **Special Notes**:
    - The released villager fully retains all original data, including trade lock states.
    - **Bed and job site bindings are cleared** upon release — the villager must find and re-bind to a new job site.
  - **JEI Support**: A full information page has been registered in JEI, detailing all usage rules of the Villager Cage.
  - **Creative Mode**: Added to the "Tuanzi's Mod" creative mode item group.
- **New Block/Item: Soul Merchant Station**:
  - **Crafting Recipe**: Shaped crafting, pattern as follows:
    ```
    Nether Quartz      Sculk Block      Nether Quartz
    Redstone Comp.     Villager Cage    Redstone Comp.
    Nether Quartz      Sculk Block      Nether Quartz
    ```
    (Can be crafted using an empty Villager Cage)
  - **Villager Import & AI-free Trading**:
    - Right-click the station with a filled Villager Cage to securely import the villager (the cage is consumed).
    - Upon importing, a **beautiful 3D rotating item representing the villager's profession** (e.g., Wheat for Farmer, Book for Librarian, Iron Pickaxe for Toolsmith) will float inside the cage, offering amazing visual aesthetics!
    - Right-clicking the station directly opens the standard trading GUI, fully supporting reputation discounts, "Hero of the Village" discounts, and villager renaming (right-click with a Name Tag).
    - Trade experience accumulation, leveling, and unlocking new trades are fully functional, completely stripping away entity AI to achieve supreme server TPS/FPS optimization!
  - **Auto-Restock via Adjacent Job Sites**:
    - No entity pathfinding required. If a matching, **unoccupied job site block** (e.g., Lectern, Brewing Stand) is adjacent (6 directions) to the station, locked trades (red cross) will automatically restock at dawn or upon waking up from a bed!
  - **Villager Export**:
    - **Sneak + Right-click** the block with an empty Villager Cage to securely package the villager back into the cage. The station resets to empty and the block remains intact.
  - **Blast Immunity**:
    - The block has an extremely high blast resistance value, **completely immune to TNT, Creepers, and all explosions**, offering permanent absolute safety to the villager.
  - **Creative Mode & JEI Support**:
    - Registered in the creative tab and has a detailed, dedicated JEI tutorial information page.
  - **New Feature: Villager Rotation Adjustment in Soul Merchant Station**:
    - When a villager is present inside the Soul Merchant Station, players can **Sneak (Shift) + Right-click the block** (with any hand except holding an empty Villager Cage) to rotate the miniature 3D villager entity **90° clockwise**.
    - This interaction triggers a unique Spawner placement sound effect, allowing players to freely adjust and lock the villager's facing orientation in their base.
    - Replaced the continuous automatic idle spinning of the villager inside, ensuring the villager's orientation remains stable, locked, and fully customizable.

- **New Item/Teleportation Book: Traveler's Notebook**:
  - **Core Mechanism**: When held in main hand or offhand, right-clicking directly opens a highly immersive crystal book-style teleportation screen. Sneak + Right-click opens the internal storage interface (3x9, 27 slots).
  - **Fuel & Charging Slot (Slot 0)**:
    - The top-left slot (Slot 0) exclusively accepts the custom charging fuel — **Teleportation Paper**.
    - When the notebook interface is closed, it automatically consumes Teleportation Paper from this slot and converts it into energy (1 paper = 1 energy point, capped at 64).
    - If the energy is fully charged, any unconsumed paper will be returned to the player's inventory, or dropped at the player's feet if the inventory is full. Slot 0 is cleared after charging.
    - The item tooltip dynamically displays the current energy: "`⏣ Energy: XX/64`".
  - **Destination Storage Slots (Slot 1-26)**:
    - The remaining 26 slots exclusively accept bound **Signpost Runes**.
    - Signpost Runes are non-stackable (max stack size of 1).
  - **Teleportation Cost & Constraints**:
    - Intra-dimension teleportation costs §a1§r energy point; cross-dimension teleportation costs §c2§r energy points. If energy is insufficient, a red action bar alert will display: "Traveler's Notebook: Insufficient Energy".
    - Teleporting triggers a **3-second channelling time**. The player must stand completely still and take no damage (from mobs, fire, lava, etc.) during this period; otherwise, teleportation is instantly cancelled with a red action bar warning.
    - Plays the classic Enderman teleport sound and generates purple portal particles upon successful channelling.
    - A short cooldown is triggered after teleportation (3 seconds for intra-dimension, 5 seconds for cross-dimension). Clicking the button during cooldown displays the remaining cooldown seconds in the action bar.
  - **Signpost Rune**:
    - Right-clicking this rune instantly binds it to the current dimension and exact coordinates. Plays a ringing chime sound and shows a success alert in the action bar.
    - Bound runes can be renamed in an Anvil. The custom name will be used as the destination title in the notebook's teleportation list. If not renamed, it defaults to showing the dimension and approximate coordinates.
  - **Item Drops on Destruction**:
    - The Traveler's Notebook cannot be enchanted or repaired in an anvil.
    - If the notebook is dropped as an `ItemEntity` in the world and subsequently destroyed by lava, fire, or explosions, all non-empty Signpost Runes and Teleportation Papers stored inside will burst out at its location!

## Bug Fixes

- **Fixed Villager Cage opening Villager trade UI instead of capturing**:
  - **Cause**: The default `mobInteract` method in `Villager` (which opens the trading screen) has a higher interaction priority, intercepting and overriding the item's `interactLivingEntity` method.
  - **Fix**: Implemented a Mixin (`VillagerMixin.java`) to intercept `Villager.mobInteract` at `HEAD`. If the player holds an empty "Villager Cage", the interaction is redirected directly to the capture logic, successfully blocking the trade UI.
- **Fixed Villager Cage failing to return to "Empty" in Creative Mode**:
  - **Cause**: In creative mode, calling `stack.shrink(1)` does not actually consume or delete the item stack. Because of this, the cage retained all of its previously captured villager NBT data and CustomModelData attributes, preventing it from being used again for new captures or releases.
  - **Fix**: At the final stage of the release process, we now explicitly remove both `CUSTOM_DATA` and `CUSTOM_MODEL_DATA` from the item stack. In survival mode, the item still disappears cleanly as its count hits `0`. In creative mode, the item remains but immediately degrades back into a completely clean **Empty Villager Cage**, ready for infinite capture-and-release loops.
- **Optimized Stair Seating Interaction Mechanics**:
  - **Empty-Hand Requirement**: Now players will only trigger sitting on stairs when **both hands are completely empty** (empty-handed right-click). If either hand holds any item, it will proceed with the normal interaction or block placement, avoiding accidental seating during building or other gameplay activities.
  - **Dismount Location Optimization**: Fine-tuned the safety landing spot when players dismount from the stairs (by pressing Shift). The dismount location is now precisely set to the top surface of the stair block, preventing the passenger from sliding off or falling down from the stairs when getting up.
- **Fixed Soul Merchant Station Digging Speed & Tool Match Issue**:
  - **Pickaxe Speed Boost**: Added the Soul Merchant Station to the `minecraft:mineable/pickaxe` block tag, allowing pickaxes to mine it efficiently like a regular heavy block, fully resolving the issue where pickaxes did not speed up destruction.
  - **Mining Tool Restriction**: Tagged the block under `minecraft:needs_iron_tool` (requires Iron Pickaxe or above) to drop normally. Due to its high blast-resistant hardness setting, this effectively prevents players from accidentally breaking and losing the block using bare hands or lower-tier tools.
- **Upgraded Soul Merchant Station to a Brand-New 立体 3D Physical Model**:

  - **3D Frame Details (Golden Ratio Thicker Upgrade)**:

- **Completely resolved the 3D physical model rendering stretch and "2D Flat Item" fallback degradation bug under the new 1.21.4 (26.1) item asset system**:
  - **Cause & Fallback Mechanism**: In Minecraft 1.21.4 (26.1)'s newly overhauled Item Asset System, directly mapping the item definition file (`items/soul_merchant_station.json`) via `type: minecraft:model` to point directly to a block model (`block/soul_merchant_station`) across directories causes a sandbox/specification conflict. This triggers a **Fallback degradation render**! The game bypasses our hand-crafted 3D elements and display configurations entirely, degrading the block item into a flat, 2D textured sprite sheet based on `soul_merchant_station.png`, scaled and stretched in the viewport, resulting in a giant floating 2D sheet.
  - **Resolutions**:
    1. **Established Traditional models/item Structure**: Created a proper item model at `models/item/soul_merchant_station.json` with `"parent": "tuanzis_mod:block/soul_merchant_station"`, ensuring correct asset routing.
    2. **Updated items Definition Pointer**: Redirected `items/soul_merchant_station.json` (for both resources and generated folders to prevent Gradle overrides) to point strictly to `"tuanzis_mod:item/soul_merchant_station"`, eliminating asset parser conflicts.
    3. **Decoupled Block Elements & Display Injected**: Set the block's parent to the purest `"minecraft:block/block"` and injected the precise `"display"` scaling/translation maps into the block model JSON.
  - **Final Presentation**: Held in hands (0.4 scale), in GUI slots (0.625 scale angled at 45 degrees), dropped on the floor (0.25 rotating/floating), or framed (0.5 scale), it renders perfectly as a highly premium, transparent 3D miniature ritual cage with genuine physical depth, completely eliminating the "2D flat sheet" fallback bug.
- **Completely Fixed and Artistically Re-engineered the 3D Model & Collision Box for the Soul Merchant Station**:
  - **Premium Lightweight Dome & Skeletal Frame**: The previous draft model utilized heavy, solid 16x16x2 base and ceiling plates, along with bulky 3x3 support pillars that severely blocked internal viewport visibility.   - **Fine-grained Local UV Mapping Re-engineering**: Programmed a precision-designed `uv` coordinate array for each of the 10 elements, ensuring the stone textures, curved dome, blue columns, and inner core mesh align seamlessly without any misalignment.
  - **Floating Fusion Core Uplift**: Refactored the internal fusion core pedestal to 4x1x4 and suspended it high up at Y=5, dramatically enhancing the mysterious, floating sci-fi aesthetic.
  - **Synchronized Precise VoxelShape Bounding Boxes**: Overrode and updated the `getShape()` method in `SoulMerchantStationBlock` to match the 10-element geometry. The physical selection outline and block collisions now align beautifully with the double-stepped base, crown dome, four 1.5x1.5 pillars, and the suspended fusion core.
  - **Resolved Client Cutout RenderType Reflections**: Patched the target class path in the client loader class to net.minecraft.client.renderer.RenderType, allowing Cutout transparency settings to initialize successfully and preventing transparent column shaders from degrading into solid black box panels.
- **Fixed Trial Dummy Model Twitching High-frequency Flicker When Not Taking Damage**:
  - **Cause**: When extracting render state on the client side, the wiggle parameter `state.wiggle` was set to `(float)entity.hurtTime + partialTicks`. Since `partialTicks` oscillates continuously between `0.0` and `1.0` in the client loop, this caused `state.wiggle > 0` to be true on every frame even when `hurtTime == 0`, leading to high-frequency twitching and model jitter.
  - **Fix**: Added a strict condition `entity.hurtTime > 0` when calculating `state.wiggle`. If not taking damage, the wiggle value is forced to `0.0F`, restoring the Trial Dummy's solid, rock-steady posture.
