# Tuanzi's Mod 1.0.5.3 Changelog

## New Features & Optimizations

- **Command-Line Depth Optimization for Gacha System**:
  - **Preview Command**: Added `/cardpool preview <pool>` command, allowing players to open the Gacha Preview GUI directly by typing the active pool ID. Full Tab-completion for pool IDs is supported.
  - **Fate Anchor Commands**:
    - Added `/cardpool fate set <pool> <rarity> <item>` command, which allows players to set a target item for fate locking in a specific pool and rarity tier. Once set, the current fate value resets to zero. This command features complete Tab-completion for `pool`, `rarity` (legendary/epic/rare/uncommon/common), and `item` (filtered target item list), alongside strict validation checks.
    - Added `/cardpool fate cancel <pool> <rarity>` command, which cancels the target item of fate locking for the specified pool and rarity tier, resetting the current fate value to zero.
  - **Counter Config Completion**: Improved the administrator `/cardpool setcounter` command to support Tab-completion for the `counterType` parameter (available options: `legendary` and `epic`).
  - **History Command Overhaul**: To support administrator views and prevent command parser (Brigadier) conflicts, the `/cardpool history` command structure has been refactored into:
    - `/cardpool history`: Retains vanilla behavior, dynamically resolving the active pool history based on held items (available to all players).
    - `/cardpool history self <pool>`: Allows players to quickly check their history for a specified pool (available to all players).
    - `/cardpool history player <player> <pool>`: Allows administrators to inspect the Gacha history of a targeted player for a specified pool (administrator/OP only).

- **Normalization Enchantment Exclusiveness & Tagging**:
  - **Tag Registration**: Introduced the "Normalization Enchantments" tag (`tuanzis_mod:exclusive_normalization`) and added "Resonance Pulse", "Abyssal Rhythm", and "Buzzing Rhythm" to it.
  - **Mutual Exclusiveness**: Configured compatibility restrictions in the generator to ensure "Resonance Pulse", "Abyssal Rhythm", and "Buzzing Rhythm" are mutually exclusive and cannot be applied to the same item simultaneously.


## Bug Fixes

- **Fix for Server Shutdown Hang & Blockage**:
  - Resolved a critical issue where the Minecraft server would hang (deadlock/fail to exit) on shutdown due to non-daemon threads remaining active in the scheduler thread pools of `PlayerGachaManager` and `PoolManager`.
  - Configured Gacha-Player-Saver and Gacha-Pool-Saver thread factories to set threads as daemon threads (`setDaemon(true)`).
  - Implemented graceful shutdown logic inside the `SERVER_STOPPING` server lifecycle callback to cancel pending tasks and force-sync all dirty cache or debounced saves to disk before exiting.
