# Tuanzi's Mod 1.0.4 更新日志

## 新增功能

- **新增物品: 缚灵笼 (Villager Cage)**:
  - **合成配方**: 有序合成，图案如下：
    ```
    铁锭    绿宝石块    铁锭
    铁锭    灵魂沙      铁锭
    铁锭    铁锭        铁锭
    ```
  - **一次性使用**: 耐久值为 1，捕捉村民后笼子即销毁。
  - **捕捉村民**:
    - 主手持有缚灵笼，右键村民即可将其捕捉入笼。
    - 捕捉时播放**灵魂沙**音效，并在村民位置产生少量**末影粒子**。
    - 被捕捉的村民从世界中移除，其**职业、等级、交易项目、已交易次数、命名牌名称**等所有数据完整保存在笼子物品中。
    - 物品图标会变为**装有村民的笼子**样式（两态贴图切换）。
    - Tooltip 显示：`内含: [村民名称] - [职业]（等级X）`
  - **捕捉限制（铁傀儡保护 - 方案B精准检测）**:
    - 仅当铁傀儡正处于**主动守护状态**（在村民 16 格范围内巡逻）时，捕捉该村民会导致范围内所有铁傀儡立即对捕捉者产生仇恨，并发出警告提示。
    - 已经在追击其他目标的傀儡不会因此被打断，仅处于守护待命状态的傀儡会转换攻击目标。
  - **释放村民**:
    - **潜行 + 右键固体方块表面**，笼子消失，村民在被瞄准的方块上方生成，恢复所有捕捉前的数据。
    - 若目标位置**无固体方块**，或生成点**空间不足**（无两格高度的空气），则拒绝释放并提示玩家。
    - 空笼潜行右键不会触发任何效果。
  - **跨维度运输**: 村民数据保存在物品 NBT 中，可在任意维度释放，不受限制。
  - **特殊说明**:
    - 释放后的村民完全恢复原状，交易锁定状态保留。
    - **床与工作站绑定会清除**，村民需在新位置重新寻找并绑定工作站。
  - **JEI 支持**: 已在 JEI 中注册信息页，详细展示缚灵笼的所有使用规则。
  - **创造模式**: 已加入"团子的模组"创造模式物品栏。
- **新增方块/物品: 灵笼贸易站 (Soul Merchant Station)**:
  - **合成配方**: 有序合成，图案如下：
    ```
    下界石英    幽匿块      下界石英
    红石比较器  缚灵笼      红石比较器
    下界石英    幽匿块      下界石英
    ```
    （缚灵笼无需装有村民，空笼即可合成）
  - **村民导入与无AI交易**:
    - 手持装有村民的缚灵笼右键点击贸易站，可将村民安全转移到贸易站中（缚灵笼消耗）。
    - 导入后，贸易站正上方笼子内会浮现一个**代表该村民职业的精美3D物品旋转浮空图标**（如草帽/小麦代表农民，书本代表图书管理员，铁镐代表工具匠等），提供超高视觉感官！
    - 右键点击贸易站即可直接打开与原村民完全一致的交易GUI，支持正常的声望折扣、“村庄英雄”特殊折扣以及村民改名（手持命名牌右键）。
    - 交易的经验累积、等级提升、概率解锁新交易机制完全保留，彻底脱离实体AI，实现极致TPS/FPS优化！
  - **工作站自动刷新**:
    - 无需任何实体寻路。在方块相邻的6面（东、西、南、北、上、下）贴有该村民对应且**未被其他村民占用的有效工作站**（如讲台、酿造台）时，每天黎明或床睡觉醒来时，交易锁定红叉会自动刷新解除！
  - **村民取出**:
    - 手持空缚灵笼，**潜行 + 右键点击方块**，可无损地将村民重新封入缚灵笼中，贸易站归为空状态，方块本身不损坏。
  - **爆炸保护**:
    - 方块本身拥有超高爆炸抗性级别，**完全免疫TNT、苦力怕等一切爆炸损坏**，内部村民永久绝对安全。
  - **创造模式与 JEI**:
    - 已加入创造模式物品栏，并配备了专属的 JEI 玩法细则信息页。
  - **功能: 灵笼贸易站内村民偏转朝向微调**:
    - 当灵笼贸易站内存有村民时，玩家**潜行 (蹲下) + 右键点击方块**（手持不是空缚灵笼时），会让内部的缩小版 3D 村民实体**顺时针旋转 90°**。
    - 旋转时会播放特殊的刷怪笼放置音效（Spawner Place），方便玩家随心定义小村民的视线面向。
    - 移除了先前小村民无休止的自动缓速自转，使其面向锁定更稳定、更具定制感。

- **新增物品/传送书: 旅者手札 (Traveler's Notebook)**:
  - **核心机制**: 主手或副手持有时，右键直接打开高沉浸感的水晶书本传送界面；潜行+右键则可打开内部存储界面（3x9 共 27 格）。
  - **燃料与充能格 (Slot 0)**:
    - 界面最左上角第一格（第 0 格）限定只能放入专门的充能原料——**传送纸 (Teleportation Paper)**。
    - 关闭手札界面后，手札会自动把此格内的传送纸消耗并转化为能量（1 张纸 = 1 点能量，上限 64 点）。
    - 若能量充到上限，剩余未能消耗的传送纸会自动退回玩家背包，若背包已满则直接作为掉落物在玩家脚下生成。第 0 格在充能后清空。
    - 物品 Tooltip 动态显示当前能量：“`⏣ 能量：XX/64`”。
  - **目的地存储格 (Slot 1-26)**:
    - 剩余的 26 个格子限定只能放入已绑定的**道标符石 (Signpost Rune)**。
    - 道标符石本身堆叠上限为 1，不可堆叠。
  - **传送消耗与限制条件**:
    - 同维度传送消耗 §a1§r 点能量，跨维度传送消耗 §c2§r 点能量。若能量不足，会在 Action Bar 上显示红色警告：“旅者手札：能量不足”。
    - 传送开始后触发 **3 秒引导时间**。引导期间玩家必须保持完全静止且不能受到任何伤害（如被怪物攻击或处于火中），否则传送立即打断并在 Action Bar 红色警告提示。
    - 引导完成时播放末影人传送经典音效并产生紫色粒子。
    - 传送结束后进入短暂冷却（同维度冷却 3 秒，跨维度冷却 5 秒），冷却中再次点击按钮会在 Action Bar 提示剩余冷却秒数。
  - **道标符石 (Signpost Rune)**:
    - 右键使用时，符石会立刻与当前世界维度及精确坐标绑定，播放清脆的激活音效并在 Action Bar 提示成功。
    - 绑定的符石可以在铁砧中改名，其自定义的名称将作为列表标题在书本传送界面里显示。若未改名，则显示其世界维度名称和坐标。
  - **掉落物被毁抛洒机制**:
    - 手札本身无法被附魔或在铁砧上修复。
    - 当手札以 `ItemEntity` 物品实体形式遗落在世界上，且被岩浆、火焰或爆炸损毁时，其内部存储的全部非空道标符石和传送纸会全部在原地爆出！

## 修复与优化

- **优化 坐在楼梯上 (Stair Seating) 交互机制**:
  - **空手触发**: 现在仅在**空手**（主手和副手均不持有任何物品）右键点击楼梯时才会触发坐下。若手上持有任何物品，则正常进行物品或方块的交互，不再意外坐下，避免了日常建造或交互时的误操作。
  - **下车落点优化**: 将玩家脱离坐下（按 Shift 键）时的安全落点精准调整至楼梯正上方，避免了原先下车时玩家意外滑落或掉下楼梯的问题，确保交互过程更加流畅自然。
- **修复 试炼假人 (Trial Dummy) 在非受击状态下的模型高频抽动 Bug**:
  - **原因**: 客户端在提取渲染状态时，受击晃动 `state.wiggle` 包含了 `partialTicks` 的累加，但未前置校验 `hurtTime`。导致假人在未受击且 `hurtTime == 0` 时，模型随每帧波动的 `partialTicks` 产生了微小的高频偏转位移，在视觉上表现为持续抽动。
  - **修复**: 在提取渲染状态的 `state.wiggle` 赋值前增加 `hurtTime > 0` 的精准前置校验，未受伤时强行锁定为 `0.0F`，使试炼假人在非受击状态下恢复稳如泰山的庄严姿态。
