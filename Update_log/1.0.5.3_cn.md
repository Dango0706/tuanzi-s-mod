# Tuanzi's Mod 1.0.5.3 更新日志

## 新增与优化

- **抽奖功能指令级深度优化**：
  - **预览命令**：新增 `/cardpool preview <pool>` 指令，普通玩家可输入活跃卡池 ID 直接弹出卡池预览 GUI。支持卡池 ID 的 Tab 键补全提示。
  - **定轨命令**：
    - 新增 `/cardpool fate set <pool> <rarity> <item>` 指令，允许玩家直接通过指令对特定卡池特定稀有度的目标物品进行定轨，设定后当前的命定值会清零。此命令支持 `pool`、`rarity`（传说/史诗/稀有/优秀/普通）和 `item`（过滤后可定轨物品 ID 列表）的完整 Tab 补全提示和精确的内容有效性检查。
    - 新增 `/cardpool fate cancel <pool> <rarity>` 指令，允许玩家取消某个卡池指定品质的定轨目标，命定值也将清零。
  - **计数器配置补全**：优化管理员命令 `/cardpool setcounter`，针对其 `counterType` 参数现已支持 Tab 键快速自动提示与补全（提示选项为 `legendary` 与 `epic`）。
  - **历史记录指令升级**：为了支持管理员查看他人以及防止 Brigadier 解析冲突，重构了 `/cardpool history` 命令结构：
    - `/cardpool history`：保持原行为不变，即根据玩家手持物品智能判定卡池（普通玩家可用）。
    - `/cardpool history self <pool>`：玩家快速查看自己指定卡池的抽卡历史记录（普通玩家可用）。
    - `/cardpool history player <player> <pool>`：管理员可以跨玩家查看被查询者在指定卡池下的详细抽卡历史记录（仅限管理员/OP可用）。

- **归一化附魔互斥标签优化**：
  - **标签归类**：新增“归一化附魔”标签（`tuanzis_mod:exclusive_normalization`），并将“共振脉冲”、“深渊律动”、“蜂鸣节律”三个模组专属附魔归入该标签。
  - **互斥规则设定**：在附魔生成器中为这三个附魔建立了严格的互斥规则，使其无法同时附魔在同一件装备上，保证了附魔的平衡性。


## 问题修复

- **服务器假死与停服阻塞修复**：
  - 修复了服务器关闭时因异步线程池（`PlayerGachaManager` 与 `PoolManager` 的定时调度线程）非守护进程特性导致的 JVM 假死、无法正常关闭的严重 Bug。
  - 将 Gacha-Player-Saver 和 Gacha-Pool-Saver 线程完全配置为守护线程（`setDaemon(true)`）。
  - 在 `SERVER_STOPPING` 事件中引入了优雅的线程池 `shutdown` 与强制同步落盘流程，防止任何防抖保存任务或缓存脏数据在停机时丢失。
